Ingresaremos en el archivo ```views.py``` en el que se expondr√°n nuestros servicios. En este caso, crearemos dos peticiones de tipo GET que nos permitan acceder a la lista de restaurantes, a un restaurante exactamente y el filtrado de busqueda de coincidencias por nombre:

```
from django.views import View
from .models import Restaurant
from django.http import JsonResponse
from django.forms.models import model_to_dict


class RestaurantListView(View):
    def get(self, request):
        if('name' in request.GET):
            restaurantList = Restaurant.objects.filter(name__contains=request.GET['name'])
        else:     
            restaurantList = Restaurant.objects.all()
        return JsonResponse(list(restaurantList.values()), safe=False)


class RestaurantDetailView(View):
    def get(self, request, pk):
        restaurant = Restaurant.objects.get(pk=pk)
        return JsonResponse(model_to_dict(restaurant))

```

## ‚ö†Ô∏è Errores y soluciones

Si tu proyecto no se ejecuta correctamente:

* Recuerda encontrarte en el path correcto. 
* Tener en un funcionamiento tu ambiente de desarrollo
* Ejecutar los comandos de migraciones al realizar al alguna modificaci√≥n que no permita el funcionamiento directo mediante ```python manage.py runserver```.
* En caso de que no reconozca la palabra ```python``` en la consola, recuerda revisar hacia donde est√° apuntando tu IDE para el funcionamiento ambiente de desarrollo. Podr√≠as intentar en lugar de ```python``` la palabra reservada ```python3``` √≥ ```py```. Por ejemplo ```py manage.py runserver```

Al realizar el front bajo el framework progresivo Vue 3, este mostraba error al llamar mediante el axios cualquier petici√≥n perteneciente a la api del proyecto en Django:

```
 No Access-Control-Allow-Origin header is present on the requested resource.
```
Puedes obtener m√°s informaci√≥n de este error aqu√≠:

* https://developer.mozilla.org/es/docs/Web/HTTP/CORS/Errors/CORSMissingAllowOrigin

Si tambi√©n te sucede, debes seguir los siguientes pasos para solucionar los problemas de CORS:

Nos situamos en la consola donde estamos ejecutando nuestro proyecto de Django y colocaremos la siguiente linea:

```
 python -m pip install django-cors-headers
```

En caso de no tener pip: (En Linux)

```
apt install python-pip
```

Luego, nos situaremos en el archivo ```webDemo/settings.py``` del proyecto ```WebDemo``` y nos iremos a la secci√≥n de ```INSTALLED APPS``` para agregar las siguientes lineas al final:
```
INSTALLED_APPS = [
    ...
    'corsheaders',
    ...
]
```

Asegurate de agregar la coma al final en caso de que no lo hayas puesto al final o podr√≠a obtener un ModuleNotFoundError.
Tambi√©n deber√°s agregar una clase de middleware para escuchar las respuestas:
```
MIDDLEWARE = [
    ...
    'corsheaders.middleware.CorsMiddleware',
    'django.middleware.common.CommonMiddleware',
    ...
]
```
CorsMiddleware debe colocarse lo m√°s alto posible, especialmente antes de cualquier middleware que pueda generar respuestas como CommonMiddleware de Django o WhiteNoiseMiddleware de Whitenoise. Si no es antes, no podr√° agregar los encabezados CORS a estas respuestas. Adem√°s, si est√°s utilizando CORS_REPLACE_HTTPS_REFERER, debe colocarse antes de CsrfViewMiddleware de Django.

Listo! Ya casi estamos. Debemos ingresar abajo de la secci√≥n ```MIDDLEWARE``` la linea que permitir√° la excpeci√≥n al problema de CORS:
```
CORS_ALLOW_ALL_ORIGINS = True
```
Despu√©s de ejecutar la aplicaci√≥n en Django con √©xito, solo nos queda ejecutar Vue para realizar la petici√≥n y ver que esta se ha realizado correctamente.
